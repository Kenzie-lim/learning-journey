## What I Learned

### 1. Digit Extraction with Operators
```python
mynum = 1234
mynum // 1000       # 1 (thousands)
mynum // 100 % 10   # 2 (hundreds)
mynum // 10 % 10    # 3 (tens)
mynum % 10          # 4 (ones)
```
- `//` = integer division, `%` = remainder
- Pattern: divide to shift, mod to extract

### 2. Input Function Behavior
- `input()` executes immediately when Python reads the line
- Right side of `=` always evaluates first, then assigns to left

### 3. String Concatenation Patterns
```python
# += method (simple)
answer += str1[i]

# list + join method (performant)
answer.append(str1[i])
''.join(answer)
```
- `'separator'.join(list)` combines list into string
- Literals can call methods directly: `''.join()`, `"hello".upper()`

### 4. For Loop Patterns Beyond range()
```python
for item in [1,2,3]:           # direct iteration
for char in "hello":           # string iteration
for i, item in enumerate(lst): # index + value
for a, b in zip(list1, list2): # parallel iteration
for k, v in dict.items():      # dictionary iteration
```
- `zip()` works with multiple iterables, stops at shortest

### 5. Boolean-Integer Conversion
```python
int(True)   # 1
int(False)  # 0
True + True # 2
sum([True, False, True])  # 2 (count of True)
```
- `bool` is subclass of `int` in Python
- Arithmetic operators trigger conversion

### 6. While-Else Construct
```python
while condition:
    if found:
        break
else:
    print("loop completed without break")
```
- `else` runs only if loop ends naturally (no `break`)

## Problems I Hit

| Problem | Solution |
|---------|----------|
| `range(n)` excludes n | Use `range(n + 1)` to include n |
| `=+` vs `+=` typo | `=+` assigns +1, `+=` increments |
| Comma in f-string `f"{b},{a}"` | Removed comma: `f"{b}{a}"` |
| Nested loop causing repeated output | Restructured logic, removed unnecessary nesting |

## Deep Dive

### Operator Usage Frequency
| Operator | Frequency | Context |
|----------|-----------|---------|
| `=`, `+=` | Very high | Assignment, accumulation |
| `==`, `!=`, `<`, `>` | Very high | Conditionals |
| `and`, `or` | High | Compound conditions |
| `-=`, `*=`, `/=` | Occasional | Convenience only |

## Key Takeaways

1. `//` and `%` combo is powerful for digit manipulation
2. Python evaluates right side of `=` before assignment
3. `zip()` is cleaner than `range(len())` for parallel iteration
4. Boolean math works because `bool` inherits from `int`
5. `while-else` handles "completed vs broke out" elegantly
